## 🖱️ 交互效果检查

### 1. 选择装备
**操作**: 点击右侧任意装备卡片

✅ **预期效果**:
- 装备卡片边框变为蓝色（选中状态）
- 左侧兼容槽位显示绿色边框+光环
- 左侧不兼容槽位变灰（透明度降低）

**示例**: 点击头盔装备
- 头盔槽位：绿色高亮
- 其他槽位：灰化

### 2. 装配装备
**操作**: 选中装备后，点击绿色高亮的槽位

✅ **预期效果**:
- 装备卡片选中状态消失（边框恢复黑色）
- 槽位显示装备图标（彩色方块）
- StatusBar计数更新："已装配: 1/12 件装备 (必选: 1/2)"（如果是头盔）
- 所有槽位恢复正常状态（无高亮/灰化）

**操作**: 选中头盔装备，点击灰色的手套槽位

✅ **预期效果**:
- 底部弹出Toast通知："此装备无法装配到该槽位"
- Toast 3秒后自动消失
- 头盔装备保持选中状态（蓝色边框）
- 槽位状态不变

### 4. 装配失败 - 已满
**操作**: 
1. 装配2件饰品到饰品槽1（maxCount=2）
2. 选中第3件饰品，点击饰品槽1

✅ **预期效果**:
- Toast通知："该槽位已满（最多2件）"
- 饰品保持选中状态
- 槽位显示"满"标记

### 5. 卸载装备
**操作**: 
1. 确保没有选中任何装备（点击已选中装备取消选中）
2. 点击已装配的槽位

✅ **预期效果**:
- 槽位中的装备被移除
- 槽位变为空状态
- StatusBar计数减少

### 6. 完整性指示器

## 🐛 常见问题排查

### 问题1: 图片不显示
**症状**: 槽位或装备卡片显示空白

**检查**:
1. 打开浏览器DevTools → Network标签
2. 刷新页面
3. 查看是否有404错误（红色）
4. 检查图片路径是否正确（应该是.svg而不是.png）

**解决**: 
```bash
npx tsx scripts/check-app-status.ts
```

### 问题2: 样式不正确
**症状**: 布局混乱，没有像素风格

**检查**:
1. 打开DevTools → Console标签
2. 查看是否有CSS错误
3. 检查Tailwind CSS是否正确加载

**解决**: 检查 `src/index.css` 是否包含 `@import "tailwindcss";`

### 问题3: 点击无反应
**症状**: 点击装备或槽位没有任何反应

**检查**:
1. 打开DevTools → Console标签
2. 查看是否有JavaScript错误
3. 检查配置文件是否正确加载

**解决**: 
```bash
npm run type-check
```

### 问题4: Toast不显示
**症状**: 装配失败时没有提示

**检查**:
1. 尝试装配不兼容的装备
2. 查看Console是否有错误
3. 检查Toast组件是否正确渲染
